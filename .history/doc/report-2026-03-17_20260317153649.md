## 2. Макрос начала печати (PRINT_START)

Разработан и отлажен полный макрос подготовки принтера к печати.

**Интеграция со слайсером (OrcaSlicer):**
- Start G-code: `PRINT_START BED=[bed_temperature_initial_layer_single] EXTRUDER=[nozzle_temperature_initial_layer]`
- End G-code: `PRINT_END`
- Строки `M190`/`M109` убраны из слайсера — управление температурой перенесено в макрос

**Последовательность PRINT_START:**

| № | Действие | Детали |
|---|---------|--------|
| 1 | Запуск нагрева стола | M140 без ожидания |
| 2 | Полный хоминг G28 | Стол греется параллельно — экономия времени |
| 3 | Ожидание экструдера | M109 |
| 4 | Ожидание стола | M190 |
| 5 | Прайм сопла | 30мм филамента медленно (5мм/с) в позиции X=285 Y=0 |
| 6 | Ретракт | 2мм — убрать каплю |
| 7 | Прайм-линия проход 1 | X=0, Y=20→120, Z=0.3мм, E=18мм |
| 8 | Прайм-линия проход 2 | Смещение X=0.5мм, Y=120→20 (рядом с первым) |
| 9 | Подъём сопла | Z=2мм — безопасная высота для слайсера |

## 4. Система смены филамента

Реализованы четыре макроса, которые автоматически отображаются
как кнопки в веб-интерфейсе Mainsail:

| Макрос | Назначение |
|--------|-----------|
| `LOAD_FILAMENT [TEMP=200]` | Загрузить филамент: нагрев → 80мм экструзия → выключить нагрев |
| `UNLOAD_FILAMENT [TEMP=200]` | Выгрузить: нагрев → 5мм прайм → 80мм ретракт → выключить нагрев |
| `M600` | Смена во время печати: пауза → парковка → выгрузка → ожидание |
| `FILAMENT_LOADED` | После вставки нового: загрузка → продолжить печать |

## 5. Устранённые ошибки

### 5.1 Move exceeds maximum extrusion

```
Move exceeds maximum extrusion (0.866mm^2 vs 0.640mm^2)
```

**Причина 1:** `G92 E0` не выполнялся перед вторым проходом прайм-линии.
Klipper считал ΔE=36мм вместо 18мм, что давало сечение 0.866мм².

**Причина 2:** Лимит по умолчанию `4 × 0.4² = 0.640мм²` слишком мал
для прайм-линии с повышенной экструзией.

**Решение:**
- Добавлен `G92 E0` перед каждым проходом прайм-линии независимо
- Параметр `max_extrude_cross_section: 2.0` добавлен в конфиг экструдера

### 5.2 LED прогресс-бар — UndefinedError

```
jinja2.exceptions.UndefinedError: 'dict object' has no attribute 'progress'
```

**Причина:** `_LED_PROGRESS` вызывался из `PRINT_START` до начала
передачи файла через виртуальный SD — `print_stats.progress` ещё не
инициализирован.

**Решение:** добавлен фильтр `| default(0) | float` к обращению к прогрессу.
