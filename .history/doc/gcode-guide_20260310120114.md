# G-code и команды Klipper — Справочник по проекту

> Обучающий документ на основе реального опыта настройки CoreXY принтера.
> Все команды проверены на BTT Octopus v1.1 + EBB 42 v1.2 + Klipper.

---

## Как отправить команду

Есть три способа:

**1. Через Mainsail** (самый удобный)
- Открыть `http://192.168.2.20`
- Внизу страницы — консоль, вводить команды вручную

**2. Через Moonraker API** (из терминала / скриптов)
```bash
curl -X POST http://192.168.2.20:7125/printer/gcode/script \
     -H "Content-Type: application/json" \
     -d '{"script": "QUERY_PROBE"}'
```

**3. Через SSH на Pi** (системные операции)
```bash
ssh pi@192.168.2.20 'команда'
```

## Стандартный G-code

Стандартные команды G-code работают так же, как на любом принтере с Klipper.

### Движение

| Команда | Описание | Пример |
|---------|----------|--------|
| `G0` | Быстрое перемещение (без экструзии) | `G0 X150 Y150 F3000` |
| `G1` | Рабочее перемещение (с возможной экструзией) | `G1 Z10 F300` |
| `G28` | Хоминг (поиск нуля) | `G28` — все оси, `G28 X Y` — только XY |
| `G90` | Абсолютные координаты | После этого X150 = позиция 150 мм |
| `G91` | Относительные координаты | После этого Z10 = сдвиг на 10 мм |
| `G92` | Установить текущую позицию | `G92 E0` — обнулить экструдер |

**Важно для CoreXY:** `G28` по Z требует работающего Z probe. Без него — только `G28 X Y`.

### Экструдер

| Команда | Описание | Пример |
|---------|----------|--------|
| `G1 E100 F200` | Подать 100 мм филамента | Используется при калибровке |
| `G92 E0` | Обнулить счётчик экструдера | Перед каждым тестовым выдавливанием |
| `M82` | Абсолютный режим экструдера | Стандартный режим |
| `M83` | Относительный режим экструдера | Удобно для ручной подачи |

### Температура

| Команда | Описание | Пример |
|---------|----------|--------|
| `M104 S220` | Нагреть хотенд до 220°C (не ждать) | |
| `M109 S220` | Нагреть хотенд и ждать достижения | |
| `M140 S60` | Нагреть стол до 60°C (не ждать) | |
| `M190 S60` | Нагреть стол и ждать достижения | |
| `M106 S255` | Включить обдув (0–255) | `M106 S128` = 50% |
| `M107` | Выключить обдув | |

### Сервис

| Команда | Описание |
|---------|----------|
| `M84` | Выключить моторы (обесточить) |
| `M112` | Аварийная остановка (emergency stop) |
| `M503` | Показать текущие настройки |

## Команды Klipper (расширенные)

Klipper добавляет свои команды поверх стандартного G-code. Пишутся заглавными буквами.

---

### Диагностика и состояние

#### `QUERY_PROBE`
Показывает текущее состояние Z датчика (нажат / свободен).

```
QUERY_PROBE
# Ответ: probe: open   ← датчик не нажат, всё в порядке
# Ответ: probe: TRIGGERED  ← датчик нажат (или пин висит HIGH без датчика)
```

**В проекте:** выяснили что пин PB8 висит HIGH без подключённого датчика.
Решение: добавить `!` в конфиг — `pin: ^!EBBCan:PB8`

---

#### `QUERY_ADC NAME=<имя>`
Показывает сырое аналоговое значение (0.0–1.0) с пина термистора.

```
QUERY_ADC NAME=heater_bed
# Ответ: ADC object "heater_bed" has value 0.999237 (timestamp 56.080)
```

Расшифровка значений:
- `0.999` — пин почти на 3.3В = термистор не подключён (open circuit)
- `0.500` — середина диапазона = примерно 25°C для NTC 100K
- `0.100` — ближе к GND = горячий термистор (>100°C)

**В проекте:** QUERY_ADC показал 0.999 — диагностировали обрыв термистора стола.

---

#### `ACCELEROMETER_QUERY`
Считывает одно показание с акселерометра ADXL345. Показывает X, Y, Z в мм/с².

```
ACCELEROMETER_QUERY
# Ответ: accelerometer values (x, y, z): -9652.8, 0.0, -296.1
```

Расшифровка:
- Гравитация = 9800 мм/с² (9.8 м/с²)
- Если X ≈ -9700, Y ≈ 0, Z ≈ 0 → EBB 42 лежит горизонтально, X направлен вниз
- Шум в покое: ±150–500 мм/с² — это нормально для ADXL345 через CAN

**В проекте:** проведено 100 измерений. Результат: Z mean=-242, stddev=527 мм/с².
Вывод: шум слишком высок для использования как Z endstop без аппаратного tap detection.

---

#### `STATUS`
Показывает общее состояние принтера (готов / ошибка / homed).

```
STATUS
# Ответ: Klipper state: Ready
```

---

### Хоминг

#### `G28`
Хоминг (поиск нулевых позиций) одной или всех осей.

```
G28          ; хоминг всех осей (X → Y → Z)
G28 X        ; только X
G28 X Y      ; только XY (без Z probe)
G28 Z        ; только Z (нужен probe)
```

**Порядок хоминга в CoreXY:**
1. X и Y едут к концевикам
2. Каретка едет в `home_xy_position` (у нас 150, 150)
3. Z опускается до срабатывания probe

**Ещё не выполнено в проекте** — ждём установки Z датчика.

---

#### `SET_KINEMATIC_POSITION`
Принудительно устанавливает текущую позицию без движения. Работает только при включённом `[force_move]`.

```
SET_KINEMATIC_POSITION Z=0
# Говорит Klipper: "считай что Z сейчас равен 0"
# Мотор не движется, просто сбрасывается логическая позиция
```

**Используется когда:** нужно разрешить движение без хоминга.
После этого можно двигаться командами G1 в относительном режиме.

```gcode
SET_KINEMATIC_POSITION Z=0
G91
G1 Z10 F300    ; поднять Z на 10 мм
G90
```

---

### Выравнивание

#### `QUAD_GANTRY_LEVEL`
Автоматическое выравнивание портала по 4 угловым точкам. Исправляет перекос рамы.

```
QUAD_GANTRY_LEVEL
# Принтер зондирует 4 угла (или больше точек)
# Подстраивает каждый из 4 Z-моторов независимо
# Ответ: Making the following Z adjustments: z=0.015, z1=-0.023, z2=0.008, z3=-0.012
```

**Требует:** полный хоминг G28, работающий Z probe.
**Ещё не выполнено** — ждём Z датчика.

---

### Калибровки

#### `PID_CALIBRATE`
Автоматическая калибровка PID регулятора температуры.

```
PID_CALIBRATE HEATER=heater_bed TARGET=60
PID_CALIBRATE HEATER=extruder TARGET=220
```

Процесс:
1. Нагревает до TARGET и выше
2. Даёт остыть
3. Повторяет несколько раз (метод Циглера-Николса)
4. Выводит коэффициенты: `PID parameters: pid_Kp=57.369 pid_Ki=0.286 pid_Kd=2872.015`

После калибровки — обязательно `SAVE_CONFIG`.

**Результат для стола (в проекте):**
- Kp=57.369, Ki=0.286, Kd=2872.015

**Важно:** heater_bed должен быть описан в том же файле, что и секция `[printer]` — не в included файле. Иначе SAVE_CONFIG выдаст ошибку конфликта.

---

#### `PROBE_CALIBRATE`
Интерактивная калибровка Z offset — расстояния от сопла до срабатывания probe.

```
PROBE_CALIBRATE
# Принтер опускает сопло над probe_point
# Вы двигаете Z вручную: TESTZ Z=-0.1
# Подкладываете лист бумаги, добиваетесь правильного зазора
# ACCEPT — сохранить, ABORT — отменить
```

Команды во время калибровки:
```
TESTZ Z=-0.1    ; опустить на 0.1 мм
TESTZ Z=+0.1    ; поднять на 0.1 мм
ACCEPT          ; принять текущее значение
ABORT           ; отменить
```

**Ещё не выполнено** — ждём Z датчика.

---

#### `SHAPER_CALIBRATE`
Автоматическая калибровка input shaper через ADXL345. Уменьшает вибрации и дефекты "ringing" на печати.

```
SHAPER_CALIBRATE
# Принтер трясётся с разными частотами
# ADXL345 записывает вибрации
# Klipper выбирает оптимальный фильтр
# Ответ: Recommended shaper is mzv @ 42.4 Hz
```

**Требует:** хоминг + принтер на probe_points (150, 150, 20).
ADXL345 уже настроен и работает — можно запустить после G28.

---

#### `SAVE_CONFIG`
Записывает результаты калибровок в конец `printer.cfg`.

```
SAVE_CONFIG
# После этого в printer.cfg появляется секция:
# *SAVE_CONFIG*
# pid_kp = 57.369
# pid_ki = 0.286
# ...
```

**Важно:** перезапускает Klipper. После перезапуска принтер готов к работе с новыми значениями.

---

### Управление нагревом

#### `SET_HEATER_TEMPERATURE`
Установить целевую температуру.

```
SET_HEATER_TEMPERATURE HEATER=heater_bed TARGET=60
SET_HEATER_TEMPERATURE HEATER=extruder TARGET=220
SET_HEATER_TEMPERATURE HEATER=heater_bed TARGET=0    ; выключить
```

**Отличие от M140/M104:** более явная команда Klipper, принимает имя нагревателя текстом.

### Управление через API (из терминала)

#### Перезапуск Klipper
```bash
# Мягкий перезапуск (firmware_restart — только MCU)
curl -X POST http://192.168.2.20:7125/printer/firmware_restart

# Полный перезапуск (reloads config)
curl -X POST http://192.168.2.20:7125/printer/restart
```

#### Состояние нагревателей
```bash
curl http://192.168.2.20:7125/printer/objects/query?heater_bed
# {"result":{"status":{"heater_bed":{"temperature":23.72,"target":0.0,"power":0.0}}}}
```

#### Последние сообщения консоли
```bash
curl "http://192.168.2.20:7125/server/gcode_store?count=10"
```

## Типичные сценарии

### Первый запуск после установки датчика Z

```gcode
QUERY_PROBE          ; убедиться что probe: open
G28 X Y              ; схоминить XY
SET_KINEMATIC_POSITION Z=50   ; сказать что Z=50 (на всякий случай)
G28 Z                ; схоминить Z через probe
QUAD_GANTRY_LEVEL    ; выровнять портал
G28 Z                ; пересхоминить Z после выравнивания
PROBE_CALIBRATE      ; откалибровать Z offset
SAVE_CONFIG
```

### Диагностика термистора

```gcode
QUERY_ADC NAME=heater_bed
; 0.999 = обрыв (не подключён)
; 0.500 = ~25°C (норма при комнатной температуре)
; < 0.100 = очень горячий (> 100°C) или короткое замыкание
```

### Движение без хоминга (форс-мув)

```gcode
SET_KINEMATIC_POSITION Z=0
G91
G1 Z20 F300     ; поднять Z на 20 мм
G90
```

### Калибровка input shaper

```gcode
G28              ; полный хоминг (нужен Z probe)
SHAPER_CALIBRATE
SAVE_CONFIG
```

## Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `ADC out of range` | Температура вышла за min_temp/max_temp | Проверить термистор, временно расширить диапазон |
| `Move out of range` | Команда движения за пределы position_max/min | Проверить позицию, скорректировать команду |
| `Probe triggered prior to movement` | Probe сработал до начала зондирования | Проверить `^!` в pin, физическое состояние датчика |
| `Must home axis first` | Попытка двигаться без хоминга | G28 или SET_KINEMATIC_POSITION |
| `section 'X' conflicts with included value` | SAVE_CONFIG пытается писать в included файл | Перенести секцию из included файла напрямую в printer.cfg |
